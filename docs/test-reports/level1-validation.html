Additional Bug Fixed This Pass
FIXED Double i binding — handleInsertAction fired twice per keypress
WorldScene.ts · createInput()

The i key was bound both via a dedicated keydown-I listener and inside the generic keydown handler, so every press of i called handleInsertAction() twice. The first call built the bridge and set the hint to "Bridge activated", but the second call immediately overwrote that hint with "Bridge already built", making first-time players think the bridge was never built.

// BEFORE — generic handler also called handleInsertAction() if (event.key === 'i' || event.key === 'I') { this.handleInsertAction(); // DUPLICATE — fired on same keypress as keydown-I } // AFTER — removed; keydown-I listener is the sole entry point // NOTE: 'i' is handled exclusively via keydown-I above; do NOT add it here // to avoid double-firing handleInsertAction() on each key press.

After the fix, pressing i at the console shows "Bridge activated. Cross the river to reach the Wave 1 gate." exactly as intended — confirmed in screenshot 11.

FIXED Playthrough script overwrote screenshots 14 & 15
scripts/playthrough.mjs

A stale tail section in playthrough.mjs ran a second crate-breaking loop after the win overlay was captured, emitting files named 14-crate-1-broken.png and 15-crate-2-broken.png that silently overwrote the correct 14-flag-triggered.png and 15-win-overlay.png.

// Removed: stale crate loop after win (shots 14-17 overwrote 14-15) // await tap(page, 'x'); // await shot(page, '14-crate-1-broken', ...); // BUG: overwrote flag shot // await shot(page, '15-crate-2-broken', ...); // BUG: overwrote win shot // await shot(page, '16-all-crates-broken', ...); // await shot(page, '17-gate-open', ...);

The script now terminates cleanly after 15-win-overlay. All 15 screenshots correctly document the expected progression.
