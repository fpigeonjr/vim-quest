Bug Fixes
FIXED SEVERITY: MED BUG-02 · Bridge re-fires on every i press
WorldScene.ts · handleInsertAction()

Pressing i at the River Console rebuilt the bridge on every keypress because setTileAt is idempotent but the toast and hint fired repeatedly and the physics colliders were re-destroyed each time, risking memory churn.

BEFORE
// No guard — fires every keypress
this.setMode('insert');
for (const col of [28,29,30]) {
  setTileAt(...);
}
AFTER
private bridgeBuilt = false;

if (this.bridgeBuilt) {
  // idempotent hint only
  return;
}
this.bridgeBuilt = true;
// build bridge once

A private bridgeBuilt flag (also persisted in GameState.bridgeBuilt so it survives reloads) short-circuits the build path after the first activation. Subsequent i presses while on the console show a "bridge already built" hint.

FIXED SEVERITY: MED BUG-03 · Progress lost on page reload
state.ts · saveState / loadSavedState · BootScene.ts

The entire game state — unlocked commands, crates destroyed, gate status, bridge status — was ephemeral and reset to defaults on every page reload.

Three changes implement a full save/restore cycle:

// state.ts — new helpers
const SAVE_KEY = 'vimquest-save-v1';

export function saveState(state: GameState): void {
  localStorage.setItem(SAVE_KEY, JSON.stringify(state));
}

export function loadSavedState(): GameState | null {
  const raw = localStorage.getItem(SAVE_KEY);
  return raw ? { ...defaultGameState(), ...JSON.parse(raw) } : null;
}
  • BootScene calls loadSavedState() and falls back to defaultGameState() if no save exists.
  • WorldScene.syncState() calls saveState() on every state patch — zero extra calls needed.
  • WorldScene.restoreFromState() re-applies open gate / built bridge to the map geometry after world generation.
  • Pressing R to restart now calls clearSavedState() so the new run starts clean.
FIXED SEVERITY: LOW BUG-04 · INSERT mode visually indistinguishable
WorldScene.ts · setMode()

The speed reduction in INSERT mode (180 → 120 px/s) was imperceptible during play. The HUD label changed to [INSERT] but the player cursor looked identical, making it hard to track current mode at a glance.

// setMode() — before: no visual change
private setMode(mode: VimMode) {
  this.syncState({ mode });
  // ↓ added
  if (mode === 'insert') {
    this.player.setTint(0x88ccff); // soft blue
  } else {
    this.player.clearTint();
  }
}

The player cursor is now tinted soft blue (#88ccff) when in INSERT mode and returns to normal on Esc. The tint is applied consistently wherever setMode() is called (insert action, console activation, Esc handler).

Feature Additions
FEATURE Crate break animation + camera shake
WorldScene.ts · handleBreakCrate()

Previously the crate tile silently swapped to path with no visual or physical feedback, making it unclear whether the x command had done anything.

// 120 ms camera shake
this.cameras.main.shake(120, 0.006);

// Scale-to-zero tween on the crate image
this.tweens.add({
  targets: crateImage,
  scaleX: 0, scaleY: 0,
  angle: 45, alpha: 0,
  duration: 220, ease: 'Back.easeIn',
  onComplete: () => {
    crateImage.setScale(1).setAlpha(1).setAngle(0);
    setTileAt(..., TILE_IDS.path);
  },
});

The crate now spins and shrinks to zero over 220 ms before the tile swap occurs, and the camera shakes briefly for impact. The image properties are reset after the tween so the underlying node is clean before setTileAt replaces its texture.

FEATURE In-world directional signposts
WorldScene.ts · buildWorld()

Players had no in-world cues pointing them toward shrines, the crate puzzle, or the River Console. The only guidance was the HUD hint bar, which many players don't read carefully during exploration.

Five signpost text objects were added at strategic waypoints:

  • Near spawn (row 7) — "► Word Shrine: w b →"
  • Vertical path (row 12) — "▼ Line Shrine: 0 $"
  • South corridor entry (row 21) — "► Operator Shrine: x →"
  • Crate cluster (row 15) — "► Break crates with x (3 to unlock the gate)"
  • River bank (row 8) — "▼ River Console — Press i to build bridge"

Each sign uses a dark panel background, coloured per zone, and a small readable font (13 px Courier New) so it doesn't compete with gameplay elements.

FEATURE First-load h j k l primer
WorldScene.ts · showIntroOverlay() · state.ts

New players were still entering Cursor Meadow without a guaranteed explanation of the base movement keys. The Mentor helped, but the game allowed wandering before that lesson happened.

if (!state.introSeen) {
  this.showIntroOverlay();
}

this.syncState({
  introSeen: true,
  hint: 'Move with h j k l, then speak to the Mentor near spawn.',
});

The first entry to Level 1 now shows a blocking primer that teaches h, j, k, and l, then dismisses on any key. A persisted introSeen flag keeps the overlay from reappearing on later loads.

FEATURE River Console beacon + proximity prompt
WorldScene.ts · createConsoleBeacon() · checkConsoleProximity()

The signpost near the river helped, but the console tile itself still blended into the map. Players could reach the river bank and still miss the exact activation point.

this.consoleGlow = this.add
  .rectangle(cx, cy, TILE_SIZE + 12, TILE_SIZE + 12, 0x8ad7e8, 0.2);

this.tweens.add({
  targets: this.consoleGlow,
  alpha: 0.55, scaleX: 1.18, scaleY: 1.18,
  duration: 850, yoyo: true, repeat: -1,
});

The River Console now has a pulsing cyan beacon and a context-aware prompt. Before i unlock it reads River Console, after unlock it switches to [i] Build bridge, and after activation it settles on Bridge active.

FEATURE Procedural music + interaction sound effects
audio.ts · WorldScene.ts · UIScene.ts

Cursor Meadow was still completely silent, which made shrine unlocks, crate breaks, bridge activation, and the win state feel flatter than the visual feedback suggested.

audioManager.startOverworldLoop(this);
audioManager.playSfx(this, 'unlock');
audioManager.playSfx(this, 'crate');
audioManager.playSfx(this, 'bridge');
audioManager.playSfx(this, 'win');

A lightweight procedural soundtrack now runs in the overworld, with synthesized cues for dialogue, mode changes, shrine unlocks, crate breaks, bridge activation, and victory. The UI also adds an [M] Audio On/Off toggle that persists in save data.
