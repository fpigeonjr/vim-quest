PASS TC-01 — Basic HJKL movement
Verify all four movement keys navigate the player through open tiles and are blocked by walls, water, and crates.
  • Press l — player moves right along the northern path.
  • Press j — player moves down the vertical corridor.
  • Walk into a wall tile — player stops cleanly.
  • Walk into water river — player stops cleanly.
All four directions functional. Arcade physics collisions block all BLOCKED_TILES correctly. Player drag (1200) stops movement promptly when key released.
PASS TC-02 — Shrine visit triggers command unlock
Walk over each shrine and verify new commands appear in the HUD and toast fires.
  • Navigate to Word Shrine at tile (14,6).
  • Observe HUD commands update to include w b.
  • Observe toast "Word Shrine: unlocked w b".
  • Walk off and back — shrine does NOT re-trigger (shrineVisits set).
Shrine detection works. One-time visit flag (shrineVisits Set) prevents duplicate unlock messages. HUD refreshes via registry events.

MARKER_POINTS array [13,18,24,30,36,43] traversed correctly in both directions. Edge-case hints shown properly.

UX Improvement Recommendations
🚧
Physical Gate Blocker
The Wave 1 Gate shrine fires a hint but does not physically block the player. A determined player can walk right through it, bypassing the crate puzzle entirely.
Place wall tiles across the path at x=40 on rows 22–23. Remove them in setTileAt when gateUnlocked becomes true. This makes the puzzle mandatory, not advisory.
🗺
Directional Signposts / NPCs
After unlocking w and b, the hint says "use them on the southern marker road" — but there are no signs, arrows, or NPCs guiding the player south. Discovery is entirely by wandering.
Add an NPC character or stone sign near each shrine that points to where the newly unlocked command should be used. Even a simple directional arrow tile in the world would dramatically reduce friction.
✨
Crate Break Visual Feedback
Crates disappear silently — the tile just swaps to path with no particle effect, screen shake, or animation. The moment feels flat, especially since it's a key mechanic introduced with its own shrine.
Add a short Phaser tween that scales the crate to 0 before the tile swap, or spawn a small burst of orange rectangles as debris. A brief camera shake (scene.cameras.main.shake) would make it feel impactful.
🎨
INSERT Mode Visual Distinction
The speed difference between NORMAL (180px/s) and INSERT (120px/s) is too subtle to notice. The only indicator is the small HUD label change. New players won't understand why movement "feels different."
Apply a visible tint to the player sprite in INSERT mode (e.g. player.setTint(0x88ccff)), add a pulsing glow effect, or display a fullscreen border overlay. Make the mode difference viscerally obvious.
🔮
Console Tile Discovery
The River Console tile at (29,13) is not marked in a way that makes it clearly interactive from a distance. The world-space label "River Console" only appears when the camera scrolls near it.
Add a subtle idle animation (pulsing purple glow or blinking icon) to the console tile so it draws the eye. A floating "!" prompt when the player walks within 3 tiles would signal interactability before they're standing on it.
💾
No Save / Checkpoint System
Progress is fully lost on page refresh. If a player unlocks 3 shrines and refreshes, they start over. This significantly hurts the learning loop — players won't practice w/b if they have to re-unlock it each session.
Persist GameState to localStorage on every syncState call. Restore it in BootScene. A "Continue" button on the Title screen would complete the loop. This is already flagged in docs/tasks.md.
📖
Onboarding for HJKL
The game drops the player in with the hint "Find the movement shrine and explore the meadow." but never explains what h/j/k/l do. Players familiar with Vim will know; everyone else is lost immediately.
Show a brief intro dialogue or overlay on first load. Something like: "You are a cursor. Move with h (←) j (↓) k (↑) l (→). Press any to start." This mirrors how classic Vim tutorials introduce the concept.
🎵
Audio Feedback
The game is completely silent. Sound effects for shrine activation, crate breaks, bridge building, and mode changes would make command feedback much more satisfying.
Phaser's WebAudioSoundManager can play generated tones procedurally (no asset files needed). A short 8-bit "ding" for shrine unlock and a "crack" sound for crates are high-ROI additions.
🔑
Minimap or Progress Indicator
The world is 60×34 tiles but the camera shows a small 1.25× viewport. Players have no sense of how large the map is, where the shrines are, or how far they've progressed.
A small minimap in a HUD corner showing shrine positions (as dots) would dramatically help orientation. Even a simple "Shrines: 2/5" counter in the HUD would communicate progress.
⚡
w/b Constraint is Too Restrictive
Word-jump only works on the exact marker road row (y=22). In real Vim, w/b work anywhere in a file. This mismatch creates an unintuitive teaching moment — the command does something in the game that doesn't match real Vim.
Consider framing this explicitly: "In this area, words are the glowing markers on the path." Use in-world storytelling to justify the constraint rather than silently blocking it, so the Vim metaphor stays coherent.

Overall Verdict

Level 1 — Cursor Meadow delivers a solid vertical slice. The shrine-based unlock progression is clever and the command-as-ability metaphor works well. The tilemap, physics collisions, and HUD registry architecture are clean and extensible.

The most critical fix is BUG-01 — the Wave 1 Gate needs an actual physical blocker. Without it, the crate puzzle is optional and the level's climax has no teeth. This is a one-line tilemap change that would transform the pacing.

Beyond bugs, the biggest UX gap is discoverability. Players who don't already know Vim will spend significant time confused about where to go after each shrine unlock. Adding NPCs, signs, or arrow tiles near each shrine would double the onboarding quality without requiring any new game systems.

The aesthetic is functional but the game world could feel much more alive with some crate-break animations, console idle pulses, and audio feedback. These are all achievable within Phaser's existing API with no new assets required since textures are already generated procedurally.
